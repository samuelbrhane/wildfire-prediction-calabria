# Data Preprocessing and Analysis

This folder contains all scripts for processing the raw dataset, building the training data, and performing exploratory data analysis (EDA).

---

## Before Running

### Files you must add manually to `../1_data/raw/`:
- `WarningAreas/` — the warning zone shapefile and all associated files (`.shp`, `.dbf`, `.shx`, `.prj`)
- Required by: `01`, `02`, `03`, `11`, `12`

### Remote server dependency:
Scripts marked with 🖥️ require access to the original dataset on the remote server at `../../Calabria_dataset/InputReteGood/`. These scripts cannot be run locally. Clone the repo on the remote server where the dataset is located and run them there. All other scripts only need files from `../1_data/processed/` which are generated by the pipeline.

---

## Files and Execution Order

Run in this order:

| # | File | Description | Needs Remote |
|---|------|-------------|--------------|
| 01 | `01_warning_areas_analysis.ipynb` | Maps each grid cell to a warning zone. Produces `cell_zones.parquet` — required by almost all subsequent scripts. | 🖥️ |
| 02 | `02_climate_analysis.ipynb` | Visualizes spatial distribution of climate variables for a sample day. | 🖥️ |
| 03 | `03_target_analysis.ipynb` | Maps wildfire occurrence across zones for a sample day. | 🖥️ |
| 04 | `04_single_day_pipeline_test.ipynb` | Validates the single-day pipeline logic. Prototype of `05`. | 🖥️ |
| 05 | `05_build_yearly_dataset.ipynb` | Runs the full data pipeline for all years in parallel. Saves one CSV per year to `../1_data/processed/yearly/`. | 🖥️ |
| 06 | `06_merge_yearly_data.ipynb` | Merges all yearly CSVs into `zone_sequence_merged.csv`. This is the main dataset used for model training. | ✅ |
| 07 | `07_eda_fire_analysis.ipynb` | EDA plots for fire occurrence: daily distributions, monthly totals, yearly totals, and time series for all 8 zones and the full region. | ✅ |
| 08 | `08_eda_climate_analysis.ipynb` | EDA plots for climate variables: monthly and yearly averages for all 8 zones and the full region. | ✅ |
| 09 | `09_eda_correlation_analysis.ipynb` | Correlation heatmaps between climate variables and fire counts per zone. Saves a CSV with same-day and lag-1 correlation summary. | ✅ |
| 10 | `10_compute_spatial_grids.py` | Runs in parallel to compute yearly fire count and temperature grids from raw `.h5` files. Saves `.npy` files to `../1_data/processed/spatial_grids/`. Must be run before `11` and `12`. | 🖥️ |
| 11 | `11_spatial_maps.ipynb` | Loads precomputed `.npy` grids and produces fire frequency and mean temperature maps overlaid on zone boundaries. | ✅ |
| 12 | `12_burned_area_analysis.ipynb` | Computes burned area statistics per zone across all years. Produces Table 2 (mean and total burned area per zone). | ✅ |
